# Contributing to ReadAI Team Projects

Thank you for your interest in contributing! This guide applies to all repositories in the readai-team organization.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your changes (`git checkout -b feature/your-feature`)
4. Make your changes
5. Commit with clear, descriptive messages
6. Push to your fork
7. Open a Pull Request

## Development Guidelines

### Code Style

- Follow the existing code style in the project
- Use meaningful variable and function names
- Keep functions focused and concise

### Commit Messages

- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Keep the first line under 72 characters
- Reference related issues when applicable

### Pull Requests

- Fill out the PR template completely
- Link related issues
- Keep PRs focused â€” one feature or fix per PR
- Ensure all tests pass before requesting review
- Update documentation if your changes affect it

## Reporting Bugs

Use the [bug report template](https://github.com/readai-team/.github/blob/main/ISSUE_TEMPLATE/bug_report.md) when filing issues. Include:

- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Environment details

## Suggesting Features

Use the [feature request template](https://github.com/readai-team/.github/blob/main/ISSUE_TEMPLATE/feature_request.md). Describe:

- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered

## Code of Conduct

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md). Please read it before participating.

## Questions?

If you have questions, feel free to open an issue in the relevant repository or reach out through our [website](https://www.aibrary.ai).
